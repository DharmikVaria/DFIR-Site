Tooling Layer

Digital Forensics Tools Reference

Tools do not replace forensic reasoning. They accelerate acquisition, parsing, searching, correlation, and reporting. A strong examiner understands what each tool extracts, what it may miss, and how to validate important findings.

Selection Principle

Use tools according to evidence type, not because they look advanced.

Disk images, memory captures, packet captures, mobile extractions, registry hives, cloud logs, and malware samples require different toolchains. The cards below explain practical beginner use and investigation value for each tool.

Autopsy

Disk forensic suite

Platform: Windows, Linux, macOS

Purpose: Graphical platform for disk image analysis, timeline review, keyword search, file recovery, and artifact extraction.

Best used for: Best for structured examination of disk images in academic, training, and case-review workflows.

Beginner use case: Load a forensic image, generate ingest modules, review web history, deleted files, recent documents, and timelines.

Practical forensic value: Provides a beginner-friendly interface over The Sleuth Kit and helps convert raw evidence into reportable findings.

FTK

Commercial forensic suite

Platform: Windows

Purpose: Full forensic analysis platform used for indexing, searching, filtering, email analysis, and large case management.

Best used for: Best for enterprise-scale cases where indexing, review workflows, and reporting are important.

Beginner use case: Import evidence, index the case, search keywords, bookmark relevant files, and export reports.

Practical forensic value: Useful for high-volume evidence review and professional case documentation.

FTK Imager

Acquisition and preview tool

Platform: Windows

Purpose: Creates forensic images, previews evidence, exports files, and verifies hashes.

Best used for: Best for disk imaging, quick evidence preview, and hash verification before deeper analysis.

Beginner use case: Attach a drive through a write blocker, create an E01 image, and record MD5/SHA hash values.

Practical forensic value: A practical acquisition tool every beginner should know because it teaches imaging discipline.

Volatility 3

Memory forensics framework

Platform: Windows, Linux, macOS

Purpose: Analyzes RAM images for processes, network connections, registry hives, DLLs, command lines, and injected code.

Best used for: Best for malware, intrusion, and volatile evidence investigations.

Beginner use case: Run process listing, network scan, DLL listing, and suspicious memory region plugins on a captured memory image.

Practical forensic value: Reveals runtime evidence that disk analysis may miss.

Wireshark

Packet analysis tool

Platform: Windows, Linux, macOS

Purpose: Inspects packet captures, protocols, streams, files, DNS, HTTP, TLS metadata, and network behavior.

Best used for: Best for detailed packet-level network investigation and protocol learning.

Beginner use case: Open a PCAP, follow TCP streams, filter DNS or HTTP traffic, and identify suspicious endpoints.

Practical forensic value: Helps prove communication patterns and reconstruct network sessions.

NetworkMiner

Network forensic analyzer

Platform: Windows, Linux via Mono

Purpose: Extracts hosts, files, credentials, sessions, images, and metadata from PCAP files.

Best used for: Best for quick passive network forensic triage.

Beginner use case: Load a PCAP and review hosts, downloaded files, user agents, and credentials detected in traffic.

Practical forensic value: Turns packet data into investigator-friendly artifacts quickly.

Bulk Extractor

Artifact extraction tool

Platform: Windows, Linux, macOS

Purpose: Scans disk images or files for emails, URLs, credit card patterns, domains, EXIF, and other features.

Best used for: Best for rapid feature extraction from large evidence sets.

Beginner use case: Run it against an image to extract email addresses, URLs, domains, and wordlists.

Practical forensic value: Finds useful leads without requiring full filesystem parsing.

Sleuth Kit

Command-line forensic toolkit

Platform: Windows, Linux, macOS

Purpose: Provides low-level filesystem tools for listing files, recovering data, and inspecting metadata.

Best used for: Best for transparent, scriptable disk and filesystem analysis.

Beginner use case: Use fls, mactime, istat, and icat to list files, create timelines, inspect inodes, and extract content.

Practical forensic value: Builds strong fundamentals because it exposes what GUI tools automate.

RegRipper

Registry analysis tool

Platform: Windows, Linux via Perl

Purpose: Parses Windows registry hives using plugins for user activity, USB devices, persistence, and system settings.

Best used for: Best for fast registry triage and targeted artifact extraction.

Beginner use case: Run plugins against NTUSER.DAT, SYSTEM, SOFTWARE, and SAM hives.

Practical forensic value: Extracts high-value Windows evidence in readable form.

Eric Zimmerman Tools

Windows artifact parsers

Platform: Windows

Purpose: Collection of precise parsers for LNK, Jump Lists, Shellbags, Amcache, ShimCache, MFT, Evtx, and more.

Best used for: Best for modern Windows artifact parsing and timeline generation.

Beginner use case: Parse LNK files, Shellbags, Amcache, and Event Logs, then combine CSV output in Timeline Explorer.

Practical forensic value: Highly trusted in DFIR practice for detailed Windows artifact analysis.

ALEAPP

Mobile artifact parser

Platform: Windows, Linux, macOS

Purpose: Parses Android artifacts and generates readable reports from extractions and filesystem dumps.

Best used for: Best for Android application and system artifact review.

Beginner use case: Load an Android extraction and review app databases, account data, location artifacts, and reports.

Practical forensic value: Makes mobile artifact review more accessible for beginners.

Cellebrite UFED

Mobile acquisition platform

Platform: Windows

Purpose: Commercial tool for acquiring and decoding mobile devices, SIMs, and application data.

Best used for: Best for law enforcement and professional mobile forensic acquisition.

Beginner use case: Acquire a supported device and review decoded messages, calls, media, apps, and locations.

Practical forensic value: Industry-recognized platform for mobile evidence acquisition and decoding.

Guymager

Forensic imaging tool

Platform: Linux

Purpose: Graphical disk imaging tool commonly used on forensic Linux distributions.

Best used for: Best for creating RAW, E01, and AFF images with hashing on Linux.

Beginner use case: Image a suspect drive through a write blocker and verify acquisition hashes.

Practical forensic value: Useful for Linux-based forensic workstations and field imaging.

dc3dd / dcfldd

Command-line imaging tools

Platform: Linux

Purpose: Enhanced dd-style imaging tools with hashing, progress, verification, and forensic logging features.

Best used for: Best for scriptable disk imaging and controlled acquisition workflows.

Beginner use case: Create a bitstream image while calculating hashes and logging acquisition details.

Practical forensic value: Teaches low-level acquisition control and repeatability.

Hashdeep / md5deep

Hashing and file audit tools

Platform: Windows, Linux, macOS

Purpose: Computes and audits recursive file hashes for integrity verification and known-file comparison.

Best used for: Best for validating evidence exports and detecting changed files.

Beginner use case: Hash a directory before and after transfer to confirm no files changed.

Practical forensic value: Supports repeatable integrity checks across collections.

CAINE

Forensic Linux distribution

Platform: Linux live environment

Purpose: Computer Aided INvestigative Environment containing forensic tools for acquisition and analysis.

Best used for: Best for bootable forensic environments and academic lab setups.

Beginner use case: Boot CAINE, mount evidence read-only, image media, and run analysis tools.

Practical forensic value: Provides a ready forensic workstation environment for practical learning.

REMnux

Malware analysis Linux toolkit

Platform: Linux virtual machine

Purpose: Toolkit for reverse engineering, malware analysis, network simulation, document analysis, and memory support.

Best used for: Best for malware triage and malicious document or network behavior analysis.

Beginner use case: Analyze suspicious scripts, decode payloads, inspect network indicators, and emulate services.

Practical forensic value: Complements forensics by explaining malicious behavior behind artifacts.
